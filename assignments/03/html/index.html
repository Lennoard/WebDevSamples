Melhores práticas - HTML

  • Usar HTML semântico
  • Não misturar CSS com HTML
  • Fechar tags

Usar HTML semântico

Usar as tags mais espefícas do HTML para as ocasiões específicas. Nem todo bloco de conteúdo precisa ser necessariamente uma div.

Exemplos de tags estruturais: main, section, aside, nav, header, footer, article.

Correto

<footer>
 <p>Copyright © DCC Digital LTDA.</p>
</footer>

Errado

<div class="footer">
 <p>Copyright © DCC Digital LTDA.</p>
</div>

Não misturar CSS com HTML

Idealmente deve-se usar css apenas no(s) seu(s) arquivos separados. Isso ajuda a leitura e manutenção futura do código por outros desenvolvedores.

Correto

<link rel="stylesheet" href="extras.css">
<p class="red">Texto estilizado</p>

Errado

<p style="color: #FF9800; display: inline;">

Fechar tags

Todas as tags que não têm conteúdo interno devem ser fechadas. O motivo de funcionar sem o fechamento é que alguns navegadores são mais inteligentes que outros e fecharam as tags correspondentes equanto a página é carregada

Correto

<p><abbr title="Bitcoin">BTC</abbr></p>

Errado

<p><abbr title="Bitcoin">BTC</p>
