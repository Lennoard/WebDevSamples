Melhores práticas - CSS

  • Não colocar tudo em um único arquivo
  • Combinar estilos semelhantes com vírgula
  • Não usar !important

Não colocar tudo em um único arquivo

Colocar todas as regras de estilo em um único arquivo pode economizar banda de rede mas introduz desafios de manutenção.

Correto

<link rel="stylesheet" href="main.css">
<link rel="stylesheet" href="slideshow.css">
<link rel="stylesheet" href="parallax.css">

Errado

<link rel="stylesheet" href="css.css">

Combinar estilos semelhantes com vírgula

Se os estilhos forem semelhantes o suficientes é legal combinálos em um único seletor, isso pouca tempo de carregamento

Correto

h1, h2, h3, h4, h5, h6 {
 color: black;
 font-family: monospace;
}

h2.lead {
 text-decoration: underline;
}

Errado

h1, h2, h3, h4, h5, h6 {
 color: black;
 font-family: monospace;
}

h2.lead {
 color: black;
 text-decoration: underline;
 font-family: monospace;
}

Não usar !important

O uso da flag !important normalmente indica que há algo de errado com a hierarquia dos estilos ou está sendo usado um framework em que não é muito fácil a substituição dos estilos. Em todo caso, há soluções aplicáveis.

Errado

h4 {
 color: black
}

#primary-container h4.primary {
 color: red!important;
}
